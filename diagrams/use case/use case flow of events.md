Вариант использования "Зарегистрироваться" позволяет неавторизованному пользователю создать учётную запись с использованием имени, электронной почты и пароля.
### Поток события для прецедента №1 "Зарегистрироваться"
#### Основной поток:
1. Вариант использования начинается, когда неавторизованный пользователь нажимает на иконку "Мой аккаунт".
2. Приложение отображает страницу, на которой расположена форма для отправки регистрационных данных.
3. Пользователь заполняет необходимые поля.
4. Данные проверяются на соответствие требованиям, если не удовлетворяют, то выполняется альтернативный поток А1.
5. Пользователь перенаправляется на страницу авторизации.
6. Вариант использования завершается.
#### Альтернативный поток А1
1. Пользователю отображаются ошибки заполнения полей.
2. Выполняется основной поток с пункта 3.

Вариант использования "Авторизоваться" позволяет неавторизованному пользователю сменить статус на авторизованного пользователя.
### Поток события для прецедента №2 "Авторизоваться"
#### Основной поток:
1. Вариант использования начинется, когда неавторизованный пользователь нажимает на иконку "Мой аккаунт" или пытается выполнить действие, доступное только для авторизованного пользователя.
2. Приложение выводит страницу, на которой расположена форма для ввода электронной почты и пароля, а также кнопки Log in.
3. Неавторизованный пользователь вводит свои email и пароль.
4. Неавторизованный пользователь нажимает кнопку Log in.
5. Сервер проверяет введённые данные. Если они неверны, выполняется альтернативный поток А2 либо А3.
6. Неавторизованный пользователь переводится в разряд авторизованных.
7. Пользователь перенаправляется на главную страницу.
8. Вариант использования завершается.
#### Альтернативный поток А2
1. Пользователю на странице входа отображается сообщение об ошибке, где говорится о том, что некорректно введён email либо его нет в БД.
2. Выполняется основной поток, начиная с пункта 2.
#### Альтернативный поток А3
1. Пользователю на странице входа отображается сообщение об ошибке, где говорится о том, что некорректно введён пароль.
2. Выполняется основной поток, начиная с пункта 2.

### Поток события для прецедента №3 "Просмотреть категории рецептов"
#### Основной поток:
1. Вариант использования начинается, когда пользователь переходит на главную страницу и нажимает на "Recipes".
2. Пользователю отображается страница, содержащая категории рецептов.
3. Пользователь может выбрать интересующую его категорию, нажав на неё, после чего он будет перенаправлен на другую страницу - поток события для прецедента №4 "Просмотреть рецепты определённой категории".
4. Вариант использования завершается.

Дополнительно пользователь может совершить поиск категорий (А4), вернуться назад (А5) либо на главную страницу (А6), перейти в аккаунт (Поток события для прецедента №8). 
#### Альтернативный поток А4
1. Пользователю предоставляется возможность найти интересующую его категорию при нажатии на строку поиска.
2. Отображение интересующей категории.
2. Выполняется основной поток, начиная с пункта 3.
#### Альтернативный поток А5
1. Пользователю предоставляется возможность возвращения к предыдущей странице при нажатии на стрелку возврата.
2. Выполняется основной поток того события, которое являлось предыдущим.
#### Альтернативный поток А6
1. Пользователю предоставляется возможность возвращения к главной (домашней) странице при нажатии на иконку "Дом".
2. Отображается главный экран веб-приложения.

### Поток события для прецедента №4 "Просмотреть рецепты определенной категории"
#### Основной поток:
1. Вариант использования начинается, когда пользователь переходит на страницу рецептов выбранной категории.
2. Пользователю отображается страница, содержащая список репептов выбранной категории.

Дополнительно пользователь может совершить поиск категорий (А4), вернуться назад (А5) либо на главную страницу (А6), перейти в аккаунт (Поток события для прецедента №8). 

### Поток события для прецедента №5 "Просмотреть рецепт"
#### Основной поток:
1. Вариант использования начинается, когда пользователь переходит на страницу выбранного рецепта.
2. Пользователю отображается страница,содержащая название,фото блюда,время приготовления, кнопки "Ingredients" и "Cooking method".
3. При нажатии кнопки "Ingredients" начнётся альтернативный поток А7.
4. При нажатии кнопки "Cooking method" начнётся альтернативный поток А8.
5. При нажатии иконки "Флаг" начнётся альтернативный поток А9.
6. Вариант использования завершается.

Дополнительно пользователь может совершить вернуться назад (А5) либо на главную страницу (А6), перейти в аккаунт (Поток события для прецедента №8). 

#### Альтернативный поток А7
1. Вариант использования начинается после нажатии кнопки "Ingredients".
2. Пользователю отображается страница, содержащая название рецепта и название вкладки, фото блюда, время приготовления, список требующихся ингредиентов и кнопка "Create a shopping list".
3. При нажатии иконки "Флаг" начнётся альтернативный поток А9.
4. При нажатии кнопки "Create a shopping list" совершится переход к потоку события для прецедента №6 "Создание списка покупок".
5. Вариант использования завершается.

#### Альтернативный поток А8
1. Вариант использования начинается после нажатии кнопки "Cooking method".
2. Пользователю отображается страница, содержащая название рецепта и название вкладки, фото блюда, время приготовления, описание способа приготовления.
3. При нажатии иконки "Флаг" начнётся альтернативный поток А9.
4. Вариант использования завершается.

#### Альтернативный поток А9
1. Вариант использования начинается после нажатии иконки "Флаг".
2. Нажав данную иконку, пользователь сохранит тот рецепт, который на данный момент просматривает.
3. Вариант использования завершается.

### Поток события для прецедента №6 "Создание списка покупок"
#### Основной поток:
1. Вариант использования начинается, когда пользователь либо переходит на главную страницу и нажимает на "Shopping list", либо при выполнении потока события для прецедента №5 "Просмотреть рецепт"  была нажата кнопка "Create a shopping list". 
2. Пользователю отображается страница, содержащая список тех ингредиентов, которые необходимо приобрести, кнопки "Add", "Delete", "Print", "Save".
3. При нажатии кнопок выполняются альтернативные потоки А10, А11, А12, А13 соответственно.
4. Вариант использования завершается.

Дополнительно пользователь может вернуться назад (А5) либо на главную страницу (А6), перейти в аккаунт (Поток события для прецедента №8). 

#### Альтернативный поток А10
1. Вариант использования начинается после того, как пользователь нажмёт на пустое пространство в окне списка покупок.
2. Пользователю предоставляется возможность вписать дополнительно какие-либо продукты и т.д.
3. После нажатия кнопки "Add" продукт добавится в список.
4. Выполняется основной поток, начиная с пункта 2.

#### Альтернативный поток А11
1. Вариант использования начинается после того, как пользователь расставит галочки напротив тех продуктов, которые хочет удалить.
2. После того, как пользователь выбрал продукты для удаления и нажал кнопку "Delete", продукт удалится из списка.
3. Выполняется основной поток, начиная с пункта 2.

#### Альтернативный поток А12
1. Вариант использования начинается после нажатия кнопки "Print". 
2. Пользователь сможет сохранить/распечатать получившийся список покупок.
3. Вариант использования завершается.

#### Альтернативный поток А13
1. Вариант использования начинается после нажатия кнопки "Save". 
2. Пользователь сможет сохранить в своём аккаунте получившийся список покупок.
3. Вариант использования завершается.

### Поток события для прецедента № 8 "Создание собственного рецепта"
#### Основной поток:
1. Вариант использования начинается, когда авторизованный пользователь нажимает на иконку "Мой аккаунт".

4. При нажатии на окна "Create recipe", "Recipes", "Shopping list" выполняются потоки событий для прецедентов №7 , №3 , № 6.
5. После появления окон "Created recipes", "Saved recipes", "Saved lists" и нажатия на одно из них выполняются альтернативные потоки А14, А15, А16 соответственно.
6. Вариант использования завершается.

Дополнительно пользователь может вернуться назад (А5) либо на главную страницу (А6).

#### Альтернативный поток А14
1. Вариант использования начинается после нажатия на окно "Created recipes". 
2. Пользователь сможет просмотреть список созданных рецептов.
3. Вариант использования завершается.

#### Альтернативный поток А15

2. Пользователь сможет просмотреть список сохранённых рецептов.
